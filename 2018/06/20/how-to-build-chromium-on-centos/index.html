首先服务器要先能翻墙

Centos 7.0+

前置准备

安装 git

官方编译说明

Install depot_tools

depot_tools 是专门用来编译 chromium

下载下来之后，可以看到里面有很多工具

设置工具路径

Get the code

使用fetch 工具，如果提示没有 fetch，那肯定是上面路径设置问题

--no-history 参数是不需要历史版本，我只需要最新版就好

下载之后会创建 src 目录

linux 版本众多，每个不同的系统，需要的依赖不一样

Centos 的需要安装一下依赖

其他版本的可以自己根据官方文档去下载

然后再执行 runhooks 明命令，也是下载一堆依赖

Setting up the build

Chromium uses Ninja as its main build tool along with a tool called GN to generate .ninja files

成功的会显示

然后开始编译, 有快速编译的配置，还没完全弄懂各种参数的含义

编译时间会很长的，慢慢等吧。。

faq

如果fetch 那步出现问题停止了，可以使用gclient sync 命令同步，如果提示 git 冲突了，找到冲突的，删除干净，重新同步即可
